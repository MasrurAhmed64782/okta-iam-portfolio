# Project 02 — Enterprise SSO, MFA & SCIM Provisioning

## Overview

This project demonstrates enterprise-grade SSO integration using **SAML/OIDC**, adaptive **MFA policy** configuration, and automated user provisioning via **SCIM** — all implemented in Okta.

---

## Objectives

- Configure SAML and OIDC SSO integrations with claims mapping and metadata configuration
- Implement Adaptive MFA policies with group-based enforcement
- Set up SCIM provisioning to automate downstream app user lifecycle
- Configure Sign-On Policies (Okta's equivalent of Conditional Access)

---

## Technologies Used

| Tool | Purpose |
|------|---------|
| Okta App Integration (SAML 2.0) | SSO with claims mapping, metadata, certificates |
| Okta App Integration (OIDC) | Modern OAuth-based SSO |
| Okta Authenticators | MFA configuration (Okta Verify, TOTP, etc.) |
| Authentication Policies / Sign-On Policies | Conditional access rules |
| SCIM Provisioning | Automated user lifecycle to downstream apps |
